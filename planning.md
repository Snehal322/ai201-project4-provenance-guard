
1. Architecture Narrative

    When a creator submits a piece of text, it first reaches the POST /submit API endpoint. 
    The endpoint validates the request and applies rate limiting to prevent abuse or excessive automated submissions.

    If the request is valid, the text is passed to the Detection Pipeline. The pipeline evaluates the content using multiple detection signals:
        Signal 1 – Perplexity Score
        Signal 2 – Burstiness Analysis
    (Optional Stretch Feature: Stylistic Feature Analysis)

    Each signal produces an independent score describing how closely the writing resembles typical human or AI-generated text.

    The scores are then forwarded to the Confidence Scoring Module, which combines them using weighted averaging to produce:
        Final classification (Likely Human, Likely AI, or Uncertain)
        Confidence score between 0 and 1

    The classification is then passed to the Transparency Label Generator, which converts the prediction into a clear message that readers can understand.

    Next, every decision—including the original content, signal scores, confidence score, timestamp, and transparency label—is stored in the Audit Log.

    Finally, the API returns a structured JSON response containing:
        Classification
        Confidence score
        Transparency label
        Individual signal scores

    If a creator disagrees with the decision, they can submit an appeal through POST /appeal, where the appeal reason is stored, the content status changes to "Under Review", and the audit log records the appeal.

2. Detection Signals

    Signal 1 – Perplexity

    What it measures
        Perplexity measures how predictable the sequence of words is in a piece of writing.

    Why it differs between human and AI writing
        Large language models often generate highly predictable text with smoother word transitions. Human writing usually contains more unexpected word choices and irregular phrasing.

    Blind Spot
        Modern AI models can intentionally increase randomness, making AI-generated writing appear more human. Likewise, highly edited or formal human writing can also have low perplexity.


    Signal 2 – Burstiness
    
    What it measures
        Burstiness measures variation in sentence lengths throughout a document.

    Why it differs between human and AI writing
    Human authors naturally alternate between short and long sentences. 
    AI-generated writing often maintains a more uniform sentence structure.

    Blind Spot
        Some human writers intentionally use consistent sentence lengths, while advanced AI models can imitate natural burstiness.


    (Stretch Feature) Signal 3 – Stylistic Features

    What it measures: 
        Vocabulary diversity
        Average word length
        Punctuation diversity
        Repeated phrases

    Why it differs
        Humans often have more varied writing styles, while AI writing may contain repeated structures or phrases.

    Blind Spot
        Writing style differs greatly among people, making style alone an unreliable indicator.

3. False Positive Scenario

    A false positive occurs when the system incorrectly labels human-written work as AI-generated.
    Because false positives can unfairly damage a creator's credibility, the system is intentionally conservative.

    Suppose a human-written poem receives:
        AI Confidence: 0.58
        Human Confidence: 0.42

    Since the confidence is relatively low, the system does not display a strong AI attribution.

    Instead, it returns the Uncertain transparency label.

    Example label shown to readers:
        "Our system cannot confidently determine whether this content was written by a human or generated by AI."

    If the creator still disagrees with the result, they can submit an appeal through POST /appeal.

    The appeal stores:
        Content ID
        Creator's explanation
        Timestamp
        Original prediction
        Updated status = Under Review

    The audit log records both the original classification and the appeal.

    This approach minimizes harm from false positives while providing a clear review process.

4. ## API Design

a. POST /submit

Purpose : Submit content for attribution analysis.

    Request

    {
    "text": "..."
    }

    Response

    {
        "id": 101,
        "classification": "Likely Human",

        "transparency_label": "This content is likely written by a human (91% confidence).",
        "signals": {
            "perplexity": 0.84,

Purpose: Submit an appeal.

    }

    "status":"under_review"
    "message": "Appeal submitted successfully."

# Detection Signals

    Signal 1: Groq LLM

    Output: 0–1 AI probability

    Weight: 60%

    Signal 2: Stylometric Heuristics

    Output: 0–1 AI probability

    Weight: 40%

    Final confidence

        0.6 × Groq
        +
        0.4 × Stylometry


# Uncertainty Representation

    Example

    0.00–0.30
    Likely Human

    0.31–0.69
    Uncertain

    0.70–1.00
    Likely AI

    A score of 0.60 means the evidence is mixed and the system is not confident enough to make a strong attribution.


# Transparency Labels

High Confidence Human
    Likely written by a human.
    Our system found strong evidence that this content was created by a human author. This assessment is not guaranteed.

High Confidence AI
    Likely AI-generated.
    Our system found strong evidence that this content was generated using AI. This assessment is not guaranteed.

Uncertain
    Unable to determine confidently.
    The available evidence is mixed, so this content cannot be reliably classified as either human-written or AI-generated.


# Appeals Workflow

Who can appeal?
    The original creator.

Information submitted
    Content ID
    Appeal reason
    Optional evidence

System actions
    Save appeal
    Change status to Under Review
    Create audit log entry

Reviewer sees
    Original text
    Detection scores
    Final confidence
    Appeal reason
    Current status


# Edge Cases

Poems with repetitive language
    May resemble AI because of repeated phrases.

Academic writing
    Highly structured human writing may look AI-generated.

Very short text
    Not enough information for reliable stylometric analysis.

AI text edited by humans
    Could fool both detectors.
